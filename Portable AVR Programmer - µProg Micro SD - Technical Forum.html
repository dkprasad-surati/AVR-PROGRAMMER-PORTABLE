Portable AVR Programmer - µProg Micro SD

In this article, we will look at the design and features of the µProg AVR microcontroller programmer, which is easy to use, very cheap to manufacture, miniature, very fast and self-contained, using an SD memory card. This kind of programmer is useful if the target device on the microcontroller is in a hard-to-reach place and you cannot (or do not want to) connect a laptop to program or update the software of the target device.

Distinctive features and main characteristics of the programmer:
compact dimensions
high programming speed:
recording - up to 12.5 Kbps;
reading - up to 14.5 Kbps;
support for FAT16 and FAT32 file system;
support for reading, writing, verifying Flash-memory and EEPROM;
support for reading, writing, verifying Fuse-bits and Lock-bits;
support for reading/writing BIN, HEX and TXT files;
the ability to set factory settings for Fuse-bits for the target microcontroller;
organization of the user interface based on the LPH7779 graphic display with the PCD8544 controller;
animation of work and when performing operations;
standard programming connector - 6-pin Atmel ISP;
the ability to update your own software from an SD memory card;
easy to use, 4 control buttons;
auto-selection of programming speed (up to 4 MHz);
operating voltage 3V, support target microcontrollers with supply voltage from 3V to 5V.

Supported microcontrollers
The tested microcontrollers are highlighted in green.
Note that not all microcontrollers are fully supported by the programmer - for some, only Fuse and Lock bit operations are supported. System information about supported microcontrollers is contained in the chip.db system file (stored on the SD card).

Schematic diagram of the programmer 4 buttons are used to navigate the menu and control the programmer (Left, Right, Up, Down). The programmer is designed for battery operation, but it is possible to use a proper power supply. The main power supply parameters of the programmer: nominal supply voltage: 3.3 V; maximum supply voltage: 3.6 V; minimum supply voltage: 2.7 V; current consumption during initialization: up to 100 mA*; current consumption during programming: up to 10 mA*; current consumption when navigating the menu: 5 mA; current consumption in the off state (Power Down mode): 0.1 μA.















* - current consumption depends on the used SD memory card.

The programmer works only from the voltage received from the batteries, no voltage regulators are installed. The process of programming the target microcontroller can take place when it is powered from 3 V to 5 V, while the programmer has a supply voltage of 3 V. On the one hand (from the side of the programmer), in order to protect the input line of the microcontroller of the programmer from a voltage higher than the supply voltage, the MISO input connected via a 3.3V zener diode (D1) and a resistor (R1). From the side of the target microcontroller, which has a supply voltage of 5 V, a voltage level of 3 V is recognized by the SCK and MOSI lines as a high level (according to the technical description, VIH=0.6VCC=0.6×5 V = 3 V is a guaranteed value).

As you have noticed, the programmer does not have a mechanical power switch, turning it on and off is done by holding the LEFT button. After turning off the power, the microcontroller turns off the power of the display and the SD card and goes into Deep Sleep mode, in which it consumes a current of 0.1 μA (typical) from the battery.

The programmer requires an SD memory card with system files and directories. The card also contains hex and bin files for flashing the target microcontroller and txt files containing the configuration byte for programming the Fuse and Lock bits. An important system file is the config.ini file, which contains the programmer's configuration data. To turn on the programmer, you need to hold the Left

button. The inclusion is signaled by the appearance of a splash screen on the display showing the current software version, which disappears after a while, after which we can use the programmer's control menu. Using the Up and Down buttons we select menu options, using the Right button we enter the selected menu item or confirm the execution of a particular operation, to the left we exit the menu item or cancel the operation.

Programmer functions (control menu):

Flash (operations with Flash-memory):
recording;
reading;
verification;

EEPROM (operations with EEPROM):
write;
reading;
verification;

Fusebits (operations with Fuse-bits):
write;
reading;
verification;
setting factory values;
Manual (Setting Bits Manually)

Lockbits (Lockbit Operations):
Write;
reading;
verification;
erasing the crystal; Settings: automatic verification; automatic erasing; show animation; show the progress of the programming process; file types. Description of Functions Write - write data to the selected memory of the target microcontroller. You must select a file from the list. When operating with Fuse- and Lock-bits, the configuration bytes are in a file with a txt extension.












Read - Read data from the selected memory type of the target microcontroller to a file. The file is created automatically in the appropriate directory on the memory card. The filename format is x.yyy, where x is the file's sequential number (For example: m8-a2 -1.hex) and yyy is the extension (bin or hex). The file name will be displayed after saving. The entire memory of the microcontroller is read, regardless of how much really significant data is. In the case of reading Fuse- and Lock-bits, the data (configuration bytes) are saved to a file with the txt extension.

Verify (verification, check) - comparison of data in the memory of the target microcontroller with the data stored in the selected file. The result of this operation is the message "Pass"(data are identical) or " Fail at x " (error at address x), where x is the address of the first byte in which a discrepancy was found in the data. In the case of Fuse- and Lock-bits, the comparison occurs with the configuration bytes in the file with the .txt extension.

Default - Reset Fuse bits to factory defaults.

Chip Erase - An internal command to erase all microcontroller memory and reset lock bits.

Description of settings (Settings menu)

Auto Verify –Enabling this option allows you to automatically check the data after writing to the memory of the target microcontroller. The value can be On (enabled) or Off (disabled). This option applies only to operations with Flash and EEPROM memory of the microcontroller, Fuse- and Lock-bits are always read after a write operation.

Auto Erase -option to apply the target microcontroller's memory erase command, always before a memory write operation. The operation is applicable only for Flash-memory of the target microcontroller. The exception is when we know for sure that the memory area we are going to write to has already been cleared and we have some data at the end of this memory (bootloader). Disabling this feature will cause the data to be overwritten without an erase operation, and if the file is properly prepared, the bootloader area will not be affected.

Show Anims (show animation) -the option enables the display of animation after each operation, depending on the result of the execution. The animation can be canceled by pressing the Left button. It is implemented by sequential playback of BMP files from an SD memory card from the corresponding directory, so users can change animation pictures on their own. BMP animation file format, resolution 84×48 pixels, monochrome images (2 bits).

Prog bar - the progress of the programming / reading / verification process. This feature can be turned on or off.

File types (file types) - allows you to select the type of file with which the programmer will work. Applicable for operations with Flash and EEPROM memory, and, depending on the choice, only HEX or BIN files will be displayed when opening the file list.

As noted earlier, the programmer requires the presence of system files on the SD memory card. The files must be placed in the appropriate directories:
files for programming Flash-memory in the "flash" folder,
EEPROM files - in the "eeprom" folder,
configuration txt files for programming Fuse-bits - in the "fuseb" folder,
Lock-bits - in the folder "lockb".

The “System” folder contains other files necessary for the programmer to work, including the microcontroller database (chip.db file), settings (config.ini file), BMP images. An archive with the contents of the SD card with all the necessary files is available for download in the download section, which contains the SD-files folder, the contents of which must be copied to the memory card.

You can also delete files from the memory card, for this you need to select the file, hold down the button to the left and press the button to the right. The file will be deleted.

The files for programming the memory of the target microcontroller are known to be generated by the compiler, there is no doubt about them, and we place them in the appropriate folder on the memory card. We only need to create configuration txt files for programming Fuse- and Lock-bits, the values ​​in these files must be in hexadecimal format.

A file with Fuse bits contains sequential data: low byte, high byte, extended byte in HEX . For example, the contents of the file for the Atmel ATtiny2313 microcontroller will be: E4DFFF. (this means that our MCU will be programmed with an internal oscillator with a clock frequency of 8mHz. The configuration file may contain comments and user information, but only the first three bytes are used by the programmer.

The structure of the Lock-bit configuration file is similar, except that only one is used bytes from a sequence.For example, if you need to program the LB0 and LB1 bits, then the file must contain the value FC (11111100).This rule also applies to Fuse bits, if we try to program a non-existent bit (write 0 into it), it's okay happen, we will get only a verification error, because this non-existent bit will always be read as 1.

When Fuse- and Lock-bits are read by the programmer, corresponding files with the same structure are created.

The list of files (when working with the programmer) is displayed not in alphabetical order, but in the order they were written to the memory card. Files are selected with the Up and Down buttons, selected with the Right button, canceled with the Left button.

The programmer supports 128 MB, 512 MB, 1 GB, 2 GB cards. File names on the SD card in DOS 8.3 format.

The target microcontroller's IPC connector is a standard 6-pin Atmel ISP connector. However, due to the fact that the programmer is battery powered, the Vcc connector pin (connector pin 5) has been reassigned to deliver an 8 MHz clock signal to the target microcontroller (in case external clocking is needed). The signal is fed through a 10 kΩ resistor, so there is no need to modify the standard programming cable.
Atmel standard ISP connector


The programmer has a mode of automatic detection of the programming speed via the in-circuit interface. 7 modes are supported with speeds of 4 MHz, 2 MHz, 1 MHz, 500 kHz, 250 kHz, 125 kHz and 62.500 kHz. Theoretically, a speed equal to ¼ of the operating frequency of the target microcontroller is selected, but this is not always the case. At each initialization of the programming mode, the programmer starts at the maximum frequency and performs a speed test, reading the signature bytes 10 times. In the event of an error, the operation speed (SCK frequency) is reduced and the initialization operation is repeated. If the signature bytes cannot be read at the lowest frequency, the programmer issues an error message “ no answer ”.


During the initial programming of the ATmega328P programmer microcontrollerthe following Fuse bits must be set: low: A2 (low byte) , high: D0 (high byte) , Ext: 07 (extended byte) ,. This means: operation from an internal 8 MHz oscillator without a divider by 8, the CLKO output is on, the Brown-out detector is set to 2.7 V, the Reset vector transition to the bootloader, the size of the bootloader area is 4 KB, the EESAVE bit is programmed.
